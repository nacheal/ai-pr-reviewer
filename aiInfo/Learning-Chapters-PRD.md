**版本：** v1.0  
**日期：** 2026-04-02  
**状态：** 待审核

---

## 1. 背景与目标

### 1.1 背景

PR Pilot 项目已开发完成，包含 AI Agent ReAct 循环、SSE 流式推送、Tool Use、GitHub API 集成等多个技术亮点。但项目目前缺乏引导性学习材料，新读者难以系统理解项目的设计意图和技术原理。

### 1.2 目标

- 为项目增加 **10 个学习章节**，以 Markdown 文档形式呈现
- 在网站内增加**学习中心页面**，用户可浏览并点击查看各章节
- 让学习者能从零开始，逐步理解并复现这个 AI Agent 项目

### 1.3 目标读者

有基础 JavaScript / React 经验，希望通过这个项目学习 AI Agent 开发的工程师或学习者。

## 4. 前端页面需求

### 4.2 章节内容页面（`/learn/:chapterId`）

**功能：**
- 渲染对应章节的 Markdown 内容
- 顶部显示章节标题和序号
- 底部提供「上一章」/「下一章」导航按钮
- 左侧或顶部提供返回「学习中心」的链接
- Markdown 渲染使用项目已有的 `react-markdown` + `remark-gfm`

**路由示例：**
- `/learn` → 学习中心
- `/learn/01` → 第 01 章
- `/learn/10` → 第 10 章

### 4.3 首页入口

在现有首页（`Home.jsx`）底部或导航栏增加一个「学习中心」入口链接，样式为次要按钮（不干扰主功能区域）。

## 5. Markdown 文件加载方案

**方案A（推荐）：直接 import 为字符串**

利用 Vite 的 `?raw` 后缀，在前端 Bundle 时将 Markdown 文件打包为字符串：

```javascript
import content from '../../doc/01-project-overview.md?raw'
```

优点：无需后端接口，部署简单；缺点：Markdown 变更需重新构建。

**方案B：运行时 fetch**

将 doc 文件夹放入 `public/`，前端通过 `fetch('/doc/01-project-overview.md')` 动态加载。

优点：内容可独立更新；缺点：需要处理加载状态和错误。

> **建议选择方案A**，与现有项目架构更一致，构建后直接可用。

|------|------|

## 9. 待确认事项

以下问题需要确认后才能开始开发：

1. **Markdown 加载方案**：使用方案 A（import ?raw）还是方案 B（fetch from public）？
2. **章节语言**：章节内容使用中文还是英文？（建议中文，与项目目标读者一致）
3. **导航栏**：是否需要修改现有导航栏，还是仅在首页加入口？
4. **章节深度**：每章内容是面向"快速了解"（轻量）还是"深入学习"（含练习题）？
